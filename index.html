🚀 Deploy & Cloud Sync
Host on GitHub Pages free forever · Sync data to Supabase permanently
🐙
GitHub Pages — Free Forever
Your journal lives at yourusername.github.io/fx-journal
1
Download this file — click the download button above. Rename it to index.html
2
Create a GitHub account at github.com if you don't have one. It's free.
3
Create a new repository — click the + icon → New repository. Name it fx-journal. Set it to Public. Click Create.
4
Upload your file — on the repository page, click Add file → Upload files. Drag your index.html in. Click Commit changes.
5
Enable GitHub Pages — go to repository Settings → Pages. Under Source, select Deploy from a branch. Choose main branch, / (root) folder. Click Save.
6
Wait ~60 seconds then visit:
https://YOURUSERNAME.github.io/fx-journal
Replace YOURUSERNAME with your GitHub username
!
Embed in Notion: In Notion type /embed, paste your GitHub Pages URL, press Enter. Your journal appears inside Notion.
When you update the file — just re-upload index.html to GitHub and it auto-deploys in ~30 seconds.
⚡
Netlify Drop — Faster (30 seconds)
Drag and drop. No account needed.
1. Go to app.netlify.com/drop
2. Drag your index.html onto the page
3. Get a URL like random-name.netlify.app
4. Embed in Notion with /embed
⚠ Free Netlify sites expire after a while without an account. GitHub Pages is permanent.

Supabase — Permanent Cloud Storage
Free PostgreSQL database · Data syncs across all devices
✓ Free tier: 500MB database, unlimited reads/writes
✓ Data persists even if you clear browser cache
✓ Same data on phone, laptop, any browser
✓ Works with Notion embed too
Setup (one time, ~3 minutes)
1
Go to supabase.com → Start for free → Create a new project. Choose any name and region closest to you.
2
In your project, go to SQL Editor and run this query to create the trades table:
create table if not exists fxj_data (
  id text primary key,
  data jsonb not null,
  updated_at timestamptz default now()
);
alter table fxj_data enable row level security;
create policy "public access" on fxj_data for all using (true); Click to copy SQL ↗
3
Go to Project Settings → API. Copy your Project URL and your Publishable key (starts with sb_publishable_). If you see a legacy anon key starting with eyJ that works too.
4
Paste them below and click Connect:

How it works
💾
Auto-saves locally first — every trade always saves to localStorage instantly, even offline.
☁️
Syncs to Supabase — when connected, every save auto-pushes to your cloud database too.
📱
Any device — open the journal on your phone, click Pull from Cloud, and all your trades appear.
🔒
Your data, your database — data lives in your own Supabase project, not on any third-party server.
